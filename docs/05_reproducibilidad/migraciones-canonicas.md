# Migraciones Canonicas

## Politica
- Todo cambio breaking al esquema canonico requiere incremento mayor de version.
- Todo cambio aditivo que afecte lectura o materializacion debe actualizar el contrato maquina-legible.
- Ninguna migracion puede asumir que los derivados son fuente de verdad.

## Registro actual
- `1.0.0`: baseline desktop-first para cierre B0 desde escritorio.
