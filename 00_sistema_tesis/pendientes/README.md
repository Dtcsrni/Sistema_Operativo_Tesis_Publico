# Tracker de Issues — OpenClaw / Tesis de Posgrado

Directorio de issues locales gestionados por el skill `triage`.

## Convenciones

**Naming**: `YYYY-MM-DD_<category>_<slug>.md`

**Frontmatter requerido**:
```yaml
---
title: "<título>"
date: YYYY-MM-DD
category: bug | enhancement
status: needs-triage | needs-info | ready-for-agent | ready-for-human | wontfix
reporter: "<nombre>"
---
```

**Estados**:
| Estado | Descripción |
|---|---|
| `needs-triage` | Pendiente de evaluación |
| `needs-info` | Esperando información del reporter |
| `ready-for-agent` | Especificado, listo para agente AFK |
| `ready-for-human` | Requiere implementación humana |
| `wontfix` | No se accionará |

## Uso

Invocar el skill `triage` para gestionar este directorio.
Todo output del agente durante triage debe comenzar con: `> *This was generated by AI during triage.*`
